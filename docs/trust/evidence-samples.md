| Artefacto | Vertical | Objetivo |
| :--- | :--- | :--- |
| `finance-incident-pack-anon.json` | Banca/seguros | Respuesta de incidente trazable (<24h). |
| `legal-expedient-hashlog-anon.json` | Asesorías/legaltech | Trazabilidad de expediente por hash y versión. |
| `ot-change-log-anon.csv` | Industria/OT | Control de cambios operativos auditable. |
| `public-procurement-evidence-anon.json` | Sector público | Evidencia de ejecución y control de expediente. |

## Uso recomendado

1. Adjuntar en comité de dirección/compliance como muestra de formato.
2. Revisar campos mínimos exigidos por cada vertical.
3. Acordar con el cliente el esquema final (retención, clasificación, firma, export).

## Límites de esta publicación

- Los archivos publicados son ejemplos sintéticos.
- No incluyen datos personales ni operativos reales.
- El esquema productivo final depende de normativa y gobierno de datos del cliente.
